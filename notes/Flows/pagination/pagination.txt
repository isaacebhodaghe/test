Consideration

1. Should the pagination button be a link always or should default as a button and have option for link

    Solution
        * if link prop isn't provided it should default to button.
         <Pagination
           {...props}
           link={({pageIndex})=>'https://.../${pageIndex}'}
        />


2. Should Toast component for LoadMore be embedded in the Pagination component 

    Solution -- 

        -- IF YES 
        * give a callback function that is called when the toast is closed. This way project team can handle
            focus behavior on that call.
        
         <Pagination
           {...props}
           onAlertClosed={()=>{}}
        /> 

        -- IF NO
        * when onChange() is called the project team would handle the logic to show the Toast Component 
        and handle the callback function from the Toast component similarly to Above solution

        <Pagination
            {...props}
            onChange={()=>{}}
        /> 
        { showToast && <Toast onClose={()=>{}}/> }
